How to read this diagram

  • Each Epic represents a distinct Jira gap theme identified from survey data
  • Stories, Sub-tasks, Spikes, and Bugs are recommended Jira work items to address each gap
  • Work types follow Montefiore's established Jira hierarchy: Epic → Story → Sub-task
  • Spikes are time-boxed investigation items; Bugs represent process defects to remediate
  • Click any Epic header to expand or collapse its child work items
  • Use the filter bar above to filter by type, severity, or search keyword

Survey signal → Jira action model

  • Each gap finding is sourced from rated questions (avg score) and/or open-text themes
  • Respondent quotes are attached directly to the relevant Story or Bug
  • Gap severity determines priority: Critical → immediate backlog, Watch → next quarter
  • This hierarchy is ready to be imported or manually created in Jira as a Gap Remediation project
